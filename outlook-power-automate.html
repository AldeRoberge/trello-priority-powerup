Guide aligné sur l’UI française. Pas de déclencheur « carte mise à jour ».

Déclencheur Trello dispo : Lorsqu'une nouvelle carte est ajoutée à un tableau. Pour les edits plus tard → 2ᵉ flux planifié.

Flux A — nouvelle carte

  1. Déclencheur : Lorsqu'une nouvelle carte est ajoutée à un tableau.
  2. Condition : échéance non vide.
  3. Créer un événement (V4) — calendrier choisi ; début/fin en yyyy-MM-dd :
formatDateTime(triggerOutputs()?['body/due'], 'yyyy-MM-dd')
formatDateTime(addDays(triggerOutputs()?['body/due'], 1), 'yyyy-MM-dd')
  1. Variable EventId = Id (contenu dynamique) de Créer un événement.
  2. Mettre à jour une carte — description :
concat(
  coalesce(triggerOutputs()?['body/desc'], ''),
  '

[outlook-event-id]: ',
  variables('EventId')
)

Flux B — récurrence

Toutes les heures : lister les cartes, créer si pas d’id, sinon mettre à jour l’événement. Nécessaire car Trello n’envoie pas « carte modifiée » à Automate.

Cerveau ne crée pas l’événement : il masque seulement [outlook-event-id] dans le popup.
